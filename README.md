# Cursor Agent Architecture

Technical reference repository for Cursor's Background Agent runtime.

The repo is organized for GitHub readers first: start with curated docs in `docs/`, then use `extracted/` for raw evidence.

## Recommended Reading Order

1. [Key Findings](docs/KEY_FINDINGS.md)
2. [Latest Findings Snapshot](LATEST_FINDINGS.md)
3. [Full Architecture Reference](docs/ARCHITECTURE_REFERENCE.md)
4. [Documentation Index](docs/README.md)
5. [Session Wave Notes (raw appendix, optional)](docs/SESSION_WAVE_NOTES.md)
6. [Raw Evidence Index (optional)](extracted/README.md)

## What You Should Expect

- `docs/` contains human-readable architecture explanations.
- `extracted/` contains raw extracted files, logs, configs, and reverse-engineering artifacts.
- `LATEST_FINDINGS.md` is an auto-generated snapshot from extracted artifacts.
- This is a research/archive repo, not Cursor's production source code.

## Repository Layout

| Path | Purpose |
| --- | --- |
| `docs/` | Curated docs for readers |
| `docs/KEY_FINDINGS.md` | Fast high-signal summary |
| `docs/ARCHITECTURE_REFERENCE.md` | Full deep-dive technical write-up |
| `docs/SESSION_WAVE_NOTES.md` | Raw wave-by-wave command/evidence appendix |
| `LATEST_FINDINGS.md` | Auto-generated latest findings summary |
| `scripts/refresh_findings.py` | Regenerates `LATEST_FINDINGS.md` from extracted data |
| `scripts/fetch_and_refresh.sh` | Fetch/pull + regenerate helper |
| `extracted/` | Raw evidence snapshots and extracted artifacts |
| `extracted/ansible/` | Desktop provisioning playbook and assets |
| `extracted/exec-daemon-code/` | Protocol/tool/prompt extraction files |
| `extracted/system/` | Runtime package/env/tool inventories |
| `extracted/binary-analysis/` | Binary reverse-engineering notes |

## Continuous Update Workflow

```bash
# One-shot: fetch + pull + regenerate LATEST_FINDINGS.md
./scripts/fetch_and_refresh.sh

# Watch mode: rerun every 5 minutes
./scripts/fetch_and_refresh.sh --watch 300
```

Generated artifact:
- `LATEST_FINDINGS.md` (auto-generated by `scripts/refresh_findings.py`)
