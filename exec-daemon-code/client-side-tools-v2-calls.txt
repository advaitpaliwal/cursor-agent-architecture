## Key Insights

### 1. Version Evolution
  Tools evolve with v2 versions: edit_file -> edit_file_v2,
  read_file -> read_file_v2, list_dir -> list_dir_v2, task -> task_v2.
  Both versions coexist for backward compatibility.

### 2. Internal Flag
  When internal=true, the tool call is NOT added to conversation history.
  Used when the backend needs to read a file to support LLM-driven tools
  without showing it as an agent action.

### 3. Streaming Tools
  is_streaming indicates the tool produces streaming output (e.g., shell commands,
  deep search). The client renders progressive updates.

### 4. Model Call ID
  model_call_id links the tool call back to the specific model inference
  that requested it. Used for attribution and debugging.

### 5. Tool Count Growth
  Field numbers go up to 72, with gaps indicating removed/deprecated tools.
  The tool surface has grown significantly from ~20 to 42+ tools.

### 6. Computer Use Integration
  Computer use (mouse, keyboard, screenshots) is a first-class tool alongside
  traditional code tools. Combined with screen recording for debugging.

### 7. Knowledge Base & Deep Search
  Knowledge base and deep search are separate capabilities:
  - KnowledgeBase: Query pre-indexed documentation/codebase
  - DeepSearch: Multi-step, agent-driven deep search with subagents

### 8. BugBot Integration
  report_bugfix_results_params connects to the BugBot system for
  reporting automated bug fix results back to the PR review system.
