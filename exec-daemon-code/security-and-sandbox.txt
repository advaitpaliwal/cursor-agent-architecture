# Security, Sandbox, and Network Policy Schemas
# Extracted from exec-daemon webpack bundle

## SandboxPolicy.Type (agent.v1)
0: TYPE_UNSPECIFIED
1: TYPE_INSECURE_NONE         # No sandboxing at all
2: TYPE_WORKSPACE_READWRITE   # Read+write within workspace
3: TYPE_WORKSPACE_READONLY    # Read-only workspace access

## SandboxPolicy Fields
- type: SandboxPolicy.Type
- network_access: optional bool
- additional_readwrite_paths: repeated string
- additional_readonly_paths: repeated string
- debug_output_dir: optional string
- disable_tmp_write: optional bool  # Excludes /tmp, /private/tmp, /var/folders/

## NetworkPolicy (agent.v1)
- version: optional uint32 (must be 1)
- default_action: DefaultAction enum
- deny: repeated string  # Highest priority - domains, wildcards, CIDR
- allow: repeated string  # Domains, wildcards, CIDR
- logging: optional NetworkPolicyLoggingConfig

## NetworkPolicy.DefaultAction

2: DEFAULT_ACTION_DENY
