# Connect-RPC Services - exec-daemon API Surface
# Discovered via /proc cmdline extraction + Connect-RPC probing
# Protocol: Connect-RPC (buf connect v1.6.1) over HTTP/1.1
# Port 26053: ExecService + ControlService (HTTP)
# Port 26054: PtyHostService (WebSocket)

## Authentication
Auth token passed via --auth-token CLI flag
Header: Authorization: Bearer <token>
Interceptor: createAuthInterceptor(authToken) on all routes

## Service 1: agent.v1.ExecService (port 26053)
### Exec (ServerStreaming)
- Input: ExecServerMessage
- Output: stream of ExecStreamElement
- ExecStreamElement is oneof:
  - exec_client_message (ExecClientMessage)
  - exec_client_control_message (ExecClientControlMessage)
- This is the MAIN agent execution endpoint
- The orchestrator calls this to run agent conversations

## Key Architecture Insights
1. ExecService.Exec is the main agent loop - orchestrator streams messages to/from
2. ControlService provides full filesystem CRUD + git + artifact management
3. PtyHostService gives full terminal access via WebSocket
4. Auth is a simple bearer token (SHA256 hex), NOT the JWT
5. The JWT (trace-auth-token) is used for tracing to api2.cursor.sh
6. requireConnectProtocolHeader is FALSE - allows calls without Connect-Protocol-Version header
7. gzip compression is accepted
