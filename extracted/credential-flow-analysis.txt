## Authentication Flow Diagram

### Desktop Client (Cursor IDE):
  User -> OAuth2 Login -> authentication.cursor.sh -> access_token + refresh_token
  access_token stored locally by credentialManager (VS Code secure storage)
  access_token -> Authorization header -> api2.cursor.sh/aiserver.v1.DashboardService/*
  api_key alternative: loginWithApiKey(apiKey) -> access_token + refresh_token

### Cloud Agent (Background Agent):
  User -> OAuth2 Login -> access_token stored in orchestrator
  Orchestrator -> pod-daemon (port 26500) -> spawns exec-daemon
  Orchestrator passes:
    - auth-token (local RPC auth) via CLI flag
    - trace-auth-token (tracing JWT) via CLI flag
  Orchestrator does NOT pass:
    - User's access_token (stays server-side)
    - User's refresh_token
    - User's API key

### Credential Manager Code (in webpack bundle, UNUSED in cloud agent)
  class CredentialManager {
    getAccessToken() -> Promise<string>
    getApiKey() -> Promise<string>
    setAuthentication(accessToken, refreshToken, apiKey) -> Promise<void>
  }

  getValidAccessToken(credentialManager, endpoint):
    1. currentToken = credentialManager.getAccessToken()
    2. If not expiring soon, return currentToken
    3. If has apiKey, try refreshTokenWithApiKey()
    4. Return currentToken as fallback

  refreshTokenWithApiKey(credentialManager, endpoint):
    1. apiKey = credentialManager.getApiKey()
    2. LoginManager.loginWithApiKey(apiKey, {endpoint})
    3. Store new accessToken + refreshToken
    4. Return new accessToken

  createAuthInterceptor(credentialManager, requestMiddleware, opts):
    Sets headers: authorization, x-ghost-mode, x-cursor-client-version, x-request-id

## Security Architecture Insights

### 1. Principle of Least Privilege
  The sandbox receives ONLY what it needs:
  - Local auth token for RPC (can't be used externally)
  - Trace JWT with limited scope (type: "exec_daemon")
  The user's access token stays in the orchestrator

### 2. Ghost Mode (Privacy)
  When ghost-mode=true:
  - Tracing is DISABLED (no data sent to api2.cursor.sh/v1/traces)
  - The trace-auth-token is passed but never used
  - Privacy mode enum: UNSPECIFIED=0, NO_STORAGE=1, NO_TRAINING=2

### 3. Token Expiration
  - Trace JWT: 7-day expiry
  - Access token: Checked with isTokenExpiringSoon() (< 300 seconds before exp)
  - Refresh flow: Only available with apiKey (BYOK)

### 4. Memory Analysis Confirms No Leakage
  Searched entire process memory of both:
  - exec-daemon (PID 354): Only trace JWT found
  - pod-daemon (PID 1, via root escalation): No access tokens found
  The access token truly never enters the sandbox.
