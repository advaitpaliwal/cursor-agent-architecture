# Credential Provider Schemas
# Extracted from exec-daemon webpack bundle

## AzureState (aiserver.v1)
class AzureState {
    apiKey: string       // field 1
    baseUrl: string      // field 2
    deployment: string   // field 3
    useAzure: bool       // field 4
}

## BedrockState (aiserver.v1)
class BedrockState {
    accessKey: string    // field 1
    secretKey: string    // field 2
    region: string       // field 3
    useBedrock: bool     // field 4
}

## User Model Settings (in streaming config)
Fields sent per request:
- model_name: string (field 1)
- api_key: string (field 2)
- enable_ghost_mode: bool (field 3)
- azure_state: AzureState (field 4)
- enable_slow_pool: bool (field 5)
- openai_api_base_url: string (field 6)
- bedrock_state: BedrockState (field 7)
- max_mode: bool (field 8)

## Git Authentication
- Supports x-access-token (GitHub) and oauth2 (GitLab) schemes
- Configures git config insteadOf for: https://, git@, ssh://, git+ssh://
- Writes ~/.config/gh/hosts.yml for gh CLI authentication
- Handles both github.com and GitHub Enterprise hostnames

## Ghost Mode / Privacy Mode
- CLI flag: --ghost-mode <boolean> (default: true)
- Disables OpenTelemetry tracing when enabled
- Controls OTEL trace initialization
- allow_command_links_potentially_unsafe field exists (RAG injection protection)

## Secret Management
- CLOUD_AGENT_INJECTED_SECRET_NAMES env var
- BackgroundComposerSecretLevel enum:
  - ENVIRONMENT_VARIABLE: Available as env var
  - INJECTED_SECRET: Securely injected at runtime
